# AlphaFactorForge 策略設計指南 (STRATEGY_GUIDE.md)

回測策略編輯器有三種模式，難度與彈性遞增。本文件說明每種模式的用法與「程式碼模式」的完整撰寫要求，方便你設計新策略。

每個策略都由兩段邏輯組成：
- **進場條件（entry）**：成立時 → 開倉。
- **出場條件（exit）**：成立時 → 平倉（多空反手模式則反向開倉）。

訊號一律在 **K 線收盤後**評估（避免未來函數）；成交價、手續費、滑點、部位大小、方向由「執行模型」區塊控制，與本文件的訊號邏輯獨立。

---

## 模式一：參數 (params)

最快上手。填入指標週期，再從下拉選單挑一組預設進/出場訊號。

**可調參數**：快線 MA、慢線 MA、EMA 週期、RSI 週期、RSI 超買/超賣；進階區另有 MACD(快/慢/訊號)、布林(週期/倍數)、ATR、隨機指標(週期/平滑)、量能 MA。

**預設訊號**：均線黃金/死亡交叉、價格穿越 EMA、價格 vs 慢線、RSI 由超賣回升/超買回落、MACD 穿越訊號線、布林軌道觸碰、隨機指標回升/回落。

適合：快速驗證經典技術訊號。

---

## 模式二：規則積木 (blocks)

自由組合多條條件，**全部成立 (AND)** 才觸發。每條規則三欄：

`[左運算元] [運算子] [右值]`

- **左運算元 / 右值**：可選任一指標名稱（見下方清單）；右值也可直接填數字（如 `30`）。
- **運算子**：`>`、`<`、`≥`、`≤`、`上穿↑`（由下往上突破）、`下穿↓`（由上往下跌破）。

範例（進場）：
- `rsi  <  30`（RSI 低於 30）
- `maFast  上穿↑  maSlow`（快線金叉慢線）
- `price  >  bbUpper`（價格突破布林上軌）

進場可加多條（例：`rsi < 30` 且 `macd 上穿↑ macdSignal`），全部成立才進場。

適合：不寫程式，但要組合多個條件。

## 模式三：程式碼 (code) — 完整撰寫要求

最大彈性。進/出場各填一段 **JavaScript 布林運算式**（回傳 true 即觸發）。系統內部以 `new Function(...變數, 'return (你的運算式);')` 執行。

### 撰寫規則
1. 必須是**單一運算式**，會被放進 `return (...)`；**不要**寫 `return`、不要用 `;`、不要寫多行陳述式或函式宣告。
2. 可用 JS 運算子：`> < >= <= === !== && || !`、括號 `()`、算術 `+ - * /`、`Math.*`。
3. 每個變數都是「**當根 K 線**」的數值；加前綴 `prev` 代表「**前一根**」的同一變數，用來偵測穿越/轉折。
4. 指標可能在序列前段為 `null`（暖身不足）；與 `null` 比較會得到 false，屬正常，不會報錯。
5. 運算式錯誤（語法錯）會在編輯器下方紅字提示，該策略訊號暫時視為不成立。

### 可用變數（當根）
| 變數 | 意義 |
|---|---|
| `price` | 收盤價（= close） |
| `open` `high` `low` | 當根開/高/低 |
| `volume` | 成交量 |
| `maFast` `maSlow` | 快/慢簡單均線（週期見參數） |
| `ema` | 指數均線 |
| `rsi` | RSI |
| `macd` `macdSignal` `macdHist` | MACD 線 / 訊號線 / 柱 |
| `bbUpper` `bbMid` `bbLower` | 布林上/中/下軌 |
| `atr` | 平均真實波幅 |
| `stochK` `stochD` | 隨機指標 K / D |
| `volMa` | 量能均線 |

對應的前一根變數：`prevPrice prevOpen prevHigh prevLow prevVolume prevMaFast prevMaSlow prevEma prevRsi prevMacd prevMacdSignal prevMacdHist prevBbUpper prevBbMid prevBbLower prevAtr prevStochK prevStochD prevVolMa`。

### 慣用寫法
- **金叉（上穿）**：`maFast > maSlow && prevMaFast <= prevMaSlow`
- **死叉（下穿）**：`maFast < maSlow && prevMaFast >= prevMaSlow`
- **RSI 由超賣回升**：`rsi > 30 && prevRsi <= 30`
- **MACD 動能轉正**：`macdHist > 0 && prevMacdHist <= 0`
- **布林通道突破 + 量能放大**：`price > bbUpper && volume > volMa * 1.5`
- **多條件複合**：`rsi < 40 && macd > macdSignal && price > ema`

### 注意
- 進出場是各自獨立的運算式，可用完全不同的指標。
- 想做「價格站上布林中軌且 ATR 放大」這類獨創條件，直接組合即可：`price > bbMid && atr > prevAtr`。
- 部位、手續費、滑點、停損停利、做多/做空，全部在「執行模型」與停損停利欄設定，**不要**寫進運算式。

## 回測時間區段

策略編輯器的「回測區間」可限定只在某段歷史上測試（指標暖身仍使用完整資料，不會有未來函數）。用途：
- 只測最近一段行情、或只測某次大波動。
- 樣本內/外驗證：先用前段調參，再用後段（未看過的資料）驗證是否仍有效，避免過度擬合。

## 範例策略（內建，學習用）

策略編輯器頂部「📚 範例策略」下拉選單，選了即套用並回測：

| 名稱 | 模式 | 邏輯 |
|---|---|---|
| 均線黃金交叉（入門） | 參數 | 快線(9)上穿慢線(21)買、下穿賣 |
| RSI 超賣反彈（均值回歸） | 參數 | RSI<30 進、>70 出，含 5% 停損 |
| MACD 動能（程式碼） | 程式碼 | MACD 上穿訊號線進、下穿出 |
| 布林通道均值回歸（積木） | 規則積木 | 觸下軌進、回中軌出，含 4% 停損 |
| 雙均線 + RSI 濾網（進階） | 程式碼 | 站上 EMA50 且 RSI>50 進、跌破 EMA50 出 |
| 布林突破 + 量能（進階） | 程式碼 | 突破上軌且量能放大 1.5 倍進、跌破中軌出 |

## 執行模型（讓回測貼近真實）

- **手續費 / 滑點**：每邊計入；可點交易所建議值一鍵帶入（如幣安現貨吃單 0.10%）。
- **部位大小 %**：用權益的百分比下單（取代 100% all-in）。
- **成交價**：當根收盤 vs 下一根開盤（後者避免未來函數）。
- **方向**：做多 / 做空 / 多空反手。
- **Bar Magnifier 盤中精算**：在每根 K 線底下抓更細子 K 線，精準判定停損 / 停利的先後順序。只在「同一根內停損和停利都被觸及」時才會與整根近似法不同；狀態列會回報改判次數。區間過大（>9000 根子 K 線）會提示縮小日期。

## 樣本外驗證（holdout 70/30）

勾選後把回測區間切成前 70%（樣本內）/ 後 30%（樣本外），分別回測並列對照淨報酬、勝率、獲利因子、回撤、交易數。
- **樣本外接近樣本內** → 較不像過度擬合，策略較穩健。
- **樣本內賺、樣本外賠** → 高度疑似過度擬合（參數是「事後諸葛」配出來的）。

---

## 參數掃描（最佳化 + 熱力圖）

策略編輯器「⊞ 參數掃描」按鈕：
- 選 1～2 個參數設範圍與間距，跑遍所有組合（上限 256 組）。
- 用紅→黃→綠熱力圖顯示每格的最佳化指標（淨報酬 / 夏普 / 獲利因子 / 勝率 / 卡瑪 / 最小回撤），最佳格描黑框。
- 一鍵「套用最佳」把最佳參數帶回策略。
- 掃描忽略盤中精算以加速；零交易的組合不會被選為最佳。
- **提醒**：掃描出的「最佳」要再用樣本外驗證，否則就是過度擬合。
